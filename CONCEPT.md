# Dochi - 컨셉 문서

## 정의

Dochi는 개인 단말을 위한 범용 LLM 인터페이스입니다. 텍스트와 음성 입출력을 모두 지원하며, 마크다운 파일 기반 설정으로 용도에 맞게 쉽게 특화할 수 있습니다.

## 이름

"도치"는 고슴도치에서 따왔습니다. 부르기 쉬운 이름이 필요했습니다.

## 핵심 목표

**하나의 인터페이스로 다양한 용도를 커버한다.**

- 가족용: 아이 대화 상대, 가족 구성원별 프로필과 기억 분리
- 개발자용: LLM 실험, 커스텀 페르소나, MCP 서버 연동
- 업무용: 웹검색 + 메모 + 미리알림을 결합한 업무 보조

같은 앱이지만 `system.md`와 도구 설정을 바꾸면 전혀 다른 어시스턴트가 됩니다.

## 설계 원칙

1. **로컬 우선** — TTS는 로컬 ONNX, STT는 온디바이스 가능, 데이터는 로컬 저장. API 키는 사용자가 직접 관리
2. **텍스트가 기본** — 음성은 옵션. 텍스트만으로도 모든 기능을 사용할 수 있어야 함
3. **쉬운 특화** — 마크다운 파일 편집만으로 페르소나와 동작을 바꿀 수 있음
4. **투명하게** — 오픈소스, 데이터 흐름이 명확
5. **확장 가능하게** — 프로토콜 기반 구조, MCP로 도구 확장

## 사용 시나리오

### 가정

- 아이들 대화 상대 (프로필별 페르소나, 기억 분리)
- 가족 공유 기억 (일정, 약속 등 자동 축적)
- 웨이크워드로 핸즈프리 음성 인터랙션

### 개발

- 다양한 LLM 프로바이더 실험 (OpenAI, Anthropic, Z.AI)
- MCP 서버 연동 테스트
- 커스텀 도구 프로토타이핑

### 업무

- 웹검색 결합 질의응답
- 미리알림/알람 연동
- 대화 기반 브레인스토밍 (기억이 누적되므로 맥락 유지)
