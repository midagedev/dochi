# UX 명세: [K-4] 오케스트레이션 콘솔 레이아웃

> 상태: UX 설계 완료
> 이슈: #487
> 적용 범위: macOS `도구` 탭 좌측 `ExternalToolListView`

---

## 1. 문제 정의

기존 좌측 패널은 세션 탐색/히스토리/오케스트레이션 제어가 단일 스크롤에 누적되어, 운영자가 아래 질문에 빠르게 답하기 어렵다.

- 지금 막힌 작업은 무엇인가?
- 지금 개입 가능한 작업은 무엇인가?
- 방금 실행한 조치 결과는 무엇인가?

본 문서는 좌측 패널을 "목록"이 아니라 "오케스트레이션 콘솔"로 재구성하기 위한 정보 구조와 상호작용 계약을 정의한다.

## 2. 목표

핵심 사용자 흐름을 1화면에서 완료한다.

1. 상태 파악
2. 조치 실행
3. 결과 확인

## 3. 레이아웃 구조

```
┌──────────────────────────────────────────────────────────┐
│ Command Bar (고정)                                       │
│ repo 범위 | 상태 필터 | 실행/상태/요약 액션             │
├──────────────────────────────────────────────────────────┤
│ Workboard (주 영역)             │ Live Monitor          │
│ Queued / Running / Review /     │ 최신 이벤트 피드      │
│ Blocked / Done                  │ (오류/거부/요약 우선) │
├──────────────────────────────────────────────────────────┤
│ Action Queue (하단 고정)                                  │
│ 최근 실행 명령/실패 사유/재시도 가능 상태                │
└──────────────────────────────────────────────────────────┘
```

## 4. 정보 우선순위

1. `Blocked` 또는 `Failing` 항목을 상단에 고정 노출한다.
2. 현재 조치 가능한 항목에서만 액션 버튼을 활성화한다.
3. `Done`/저우선 항목은 기본 접힘 상태를 허용한다.
4. 이벤트 피드는 심각도 우선(`error > warning > info`), 동일 심각도에서는 최신순으로 정렬한다.

## 5. 상호작용 규칙

### 5.1 Command Bar
- repo 범위(`전체` + repo별)
- 빠른 상태 필터(`Blocked`, `Failing`, `Needs Review`)
- 핵심 액션(`선택`, `실행`, `상태`, `요약`)

### 5.2 Workboard
- 컬럼 기반 상태 그룹: `Queued`, `Running`, `Review`, `Blocked`, `Done`
- 카드 최소 정보: `Repo`, `Branch`, `Owner/Agent`, `Current Step`, `Health`
- 카드 선택 시 모달 대신 인라인 상세로 전개한다.

### 5.3 Live Monitor
- 이벤트는 한 줄 요약 + 상대 시간 표시
- 오류/가드 거부 이벤트는 강조 스타일 적용
- 선택 항목의 관련 이벤트를 우선 노출할 수 있다.

### 5.4 Action Queue
- 최근 실행한 오케스트레이션 액션 내역 표시
- 실패 액션은 사유와 재시도 가능 여부를 함께 제공

## 6. 반응형/접근성

- 좁은 폭에서는 Workboard를 우선 유지하고 Live Monitor는 축약 모드로 전환한다.
- 키보드 포커스 순서:
  1) Command Bar
  2) Workboard 카드
  3) Live Monitor 이벤트
  4) Action Queue 액션
- 상태와 액션 버튼에 명시적 접근성 라벨을 부여한다.

## 7. 구현 매핑 (초안)

- `ExternalToolListView`
  - `orchestrationCommandBarSection`
  - `orchestrationWorkboardSection`
  - `orchestrationLiveMonitorSection`
  - `orchestrationActionQueueSection`
- 기존 세션 선택/실행 흐름(`선택`, `실행`, `상태`, `요약`)은 동작을 유지하고 배치만 변경한다.

## 8. 비목표

- tmux 런타임 엔진 변경
- 오케스트레이션 세션 선택 알고리즘 변경
- iOS/모바일 레이아웃 신규 설계

## 9. 완료 기준

- 사용자 여정 "상태 파악 → 조치 실행 → 결과 확인"이 화면 전환 없이 수행된다.
- Blocked/Failing 항목이 기본 뷰에서 항상 우선 식별된다.
- 기존 오케스트레이션 핵심 액션이 레이아웃 변경 후에도 동작한다.
